- Xcode ships Apple macro plugins as files like `libSwiftDataMacros.dylib`, so detection logic must match `*SwiftDataMacros*` rather than `SwiftDataMacros*`.
- When `swift build` succeeds under full Xcode but a custom preflight reports missing macro plugins, verify the plugin filename pattern before assuming the Xcode install is incomplete.
- Standalone Fn / Globe support needs a dedicated lower-level monitor and Input Monitoring permission; the existing AppKit global monitor path is only reliable for Command, Option, and Control.
- When Fn / Globe is selected, keep the user's selection and surface setup/readiness guidance instead of silently normalizing it away.
- Standalone Fn / Globe capture should ignore a keyDown echo for the Fn key itself and only cancel pending activation when a different key arrives; otherwise some hardware paths can classify a standalone Fn press as a combination.
- Hold-to-talk recording needs an explicit startup state machine. If key-up arrives before async mic startup finishes, cancel the recording on completion instead of letting it start with no matching stop event.
- For a Wispr-style always-available HUD, keep the state logic in a small testable view model and use a thin non-activating `NSPanel` manager for the window behavior. That preserves background-app semantics while still allowing the dock to live across Spaces and fullscreen apps.
- For a dock that should feel anchored to the macOS Dock instead of to a screen corner, size the panel from view state and position it from the current screen's `visibleFrame` midpoint plus a small bottom offset. That keeps the HUD centered above the Dock while still respecting fullscreen and multi-display setups.
- If a floating `NSPanel` is resized manually from a view-model state change, prefer embedding the dock with a plain `NSHostingView` instead of `NSHostingController`. On macOS 26, the controller path can still push window-size constraint updates during hover transitions and crash AppKit.
- Calling `panel.setFrame(_:display:animate:)` while an NSHostingView inside it is also computing a SwiftUI layout update triggers an AppKit display-cycle exception (both EXC_BREAKPOINT and SIGABRT variants). The real fix is a fixed-size panel: keep it at the maximum visual size, let SwiftUI handle all sizing/animation internally, and use a passthrough NSView with hitTest override so mouse events in transparent areas reach windows below.
- When one floating dock has to represent hover, hold-shortcut capture, and interactive recording, separate the visual mode from the transcription status. Recording state alone is not enough to distinguish a bars-only hold HUD from a full control dock.
- Do not request microphone, Accessibility, or Input Monitoring automatically at app launch. VoiceFlow is much easier to use when microphone permission is requested on the first real recording attempt, while Smart Paste and hotkey permissions stay explicit opt-ins in Settings.
- If VoiceFlow is rebuilt with ad-hoc signing, macOS can treat it as a new privacy client and re-prompt for Microphone while silently dropping Input Monitoring trust. Stable local signing fixes both the repeated mic prompt and broken global Fn / Globe trigger persistence.
- On macOS with OpenSSL 3, `security import` can reject a PKCS#12 generated with modern defaults. Export the temporary bundle with `openssl pkcs12 -export -legacy` when bootstrapping a local code-signing identity for Keychain.
- WhisperKit model downloads on macOS land under `~/Documents/huggingface/models` by default; if refresh and recorder readiness probe `Application Support` instead, the UI will show a false `Installed` state until the next refresh or transcription attempt.
- `HubApi(downloadBase:)` expects the Hugging Face base directory, not the final `models` directory. Passing `~/Documents/huggingface/models` creates installs under `.../models/models/...`, which makes post-download verification fall back to `Get` even though WhisperKit finished.
- For repo-native benchmarking against VoiceFlow's internal transcription services, put the harness in the test target and drive it with wrapper scripts. That avoids splitting the executable target into a separate library just to expose internal WhisperKit and Parakeet code paths.
